The Temptation of Frameworks

When I decided to build my portfolio, my first instinct was to reach for React or Next.js. It's what I use at work, and the ecosystem is rich. But then I asked myself: do I actually need it?

What a Portfolio Actually Needs

A portfolio is mostly static content — some text, images, and links. There's no complex state management, no real-time data, no user authentication. The interactivity is minimal: a theme toggle, some filtering, navigation.

The Benefits of Going Plain

  • Zero build step — edit a file, refresh the browser
  • Fast — no JavaScript framework overhead
  • Simple deployment — push HTML files to GitHub Pages
  • Easy to maintain — anyone can read and edit it

The Tradeoff

The main thing I gave up is component reuse. The navbar and footer are repeated across pages. But with only five pages, that's a copy-paste I can live with.

Choose the tool that fits the problem, not the one that looks best on your resume.
